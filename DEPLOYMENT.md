# Vercel PWA Deployment Guide

## Quick Deploy

### 1. Install Vercel CLI
```bash
npm install -g vercel
```

### 2. Login to Vercel
```bash
vercel login
```

### 3. Deploy to Production
```bash
npm run deploy
```

### 4. Deploy Preview
```bash
npm run deploy:preview
```

## Deployment Steps

### Step 1: Prepare for Deployment
```bash
# Build the PWA
npm run build

# Test locally
npm run preview
```

### Step 2: Deploy to Vercel
```bash
# First time setup
vercel

# Production deployment
vercel --prod

# Or use npm scripts
npm run deploy
```

### Step 3: Configure Domain (Optional)
1. Go to Vercel Dashboard
2. Select your project
3. Go to Settings → Domains
4. Add custom domain

## Environment Variables

### Required for Production
```bash
# Add to Vercel Dashboard → Settings → Environment Variables
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### Optional Variables
```bash
NODE_ENV=production
VITE_APP_ENV=production
```

## PWA Configuration

### Manifest.json
- Automatically served with correct headers
- HTTPS required for install prompts
- Icons must be accessible

### Service Worker
- Automatically generated by Vite PWA plugin
- Cached for offline functionality
- Auto-updates when deployed

## Troubleshooting

### Common Issues

#### Install Prompt Not Showing
- Check HTTPS is enabled
- Verify manifest.json is valid
- Test on different devices

#### Offline Not Working
- Check service worker registration
- Verify cache strategies
- Test network conditions

#### Build Failures
- Check environment variables
- Verify dependencies
- Check Vercel logs
